## 一、项目概述

**产品名称：** SpecSync  
**定位：** VS Code 扩展，检测 API 文档（Spec）与代码实现之间的不一致性  
**核心价值：** 在编码时实时发现问题，将"生产故障"转化为"即时修正"

### 解决的痛点
- **75%** API 与其规范不一致
- **73%** 开发者因文档不完整受阻
- 每个过时文档问题需 **2-8 小时**解决

### 差异化定位
市场上缺乏专注于"开发时文档-代码一致性检测"的工具，SpecSync 填补该空白。

## 三、技术架构

```
src/
├── parser/
│   ├── codeParser.ts          # 统一入口
│   ├── typescriptParser.ts    # TS/TSX AST分析
│   ├── pythonParser.ts        # Python支持
│   ├── javaLikeParser.ts      # Java/Kotlin支持
│   └── specParser.ts          # Spec文档解析
├── sync/
│   └── syncEngine.ts          # 同步检测引擎
├── panel/
│   ├── treeProvider.ts        # 树形面板
│   ├── treeItem.ts            # 树节点
│   └── types.ts               # 类型定义
└── extension.ts               # 入口（400行）
```

### 架构亮点
- **LanguageParser 接口**：易于扩展新语言
- **AST 深度分析**：不只是正则匹配
- **装饰器支持**：@Spec/@Validate/@Rule
